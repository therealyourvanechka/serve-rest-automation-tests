## 4. Корзина (Carrinhos)

Корзина принадлежит конкретному пользователю

У одного пользователя может быть только одна активная корзина одновременно

Когда корзина создаётся (`POST /carrinhos`), в неё кладут товары с количеством и кол-во товара на складе **сразу** уменьшается на это количество, если товара не хватает, корзина не создаётся.

С активной корзиной можно сделать:
- **Завершить покупку** (`concluir-compra`): корзина закрывается, товар остаётся списанным
- **Отменить покупку** (`cancelar-compra`): корзина закрывается, товар **возвращается** обратно на склад

После завершения или отмены корзина больше не активна, и пользователь может создать новую

### Состояния корзины

- **S0 Нет корзины** (стартовое состояние)
- **S1 Активна** (товар уже списан со склада)

### Таблица переходов состояний

| Из состояния | Условие перехода            | В состояние                   | Валидный? | Тест      |
| ------------ | --------------------------- | ----------------------------- | --------- | --------- |
| S0           | POST (остатка достаточно)   | S1                            | Да        | CR-01     |
| S0           | POST (остатка недостаточно) | S0                            | Нет       | CR-BVA-06 |
| S1           | POST (повторно)             | S1 (без изменений)            | Нет       | CR-02     |
| S1           | concluir-compra             | S0 (товар остаётся списанным) | Да        | CR-05     |
| S1           | cancelar-compra             | S0 (товар возвращён на склад) | Да        | CR-06     |
| S0           | concluir-compra             | S0                            | Нет       | CR-08     |
| S0           | cancelar-compra             | S0                            | Нет       | CR-09     |

В тестах CR-05 и CR-06  проверить именно остаток товара и статус корзины вместе, так как это взаимосвязанные вещи

### Проверки

| ID    | Что проверяем                                                                                                                    | Предусловия                                                      | Шаги                                              | Ожидаемый результат                                                                                |
| ----- | -------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- | ------------------------------------------------- | -------------------------------------------------------------------------------------------------- |
| CR-01 | Создание корзины (товара на складе хватает)<br><br>Проверить `GET /produtos/{_id}`, что остаток товара на складе уменьшился на N | Товар с остатком >= N, у юзера нет активной корзины              | `POST /carrinhos`<br>quantidade = N               | 201                                                                                                |
| CR-02 | Нельзя создать вторую корзину, пока первая активна                                                                               | У юзера уже активная корзина                                     | `POST /carrinhos` повторно                        | 400, вторая корзина не создана                                                                     |
| CR-03 | Создание корзины с несуществующим `idProduto`                                                                                    | -                                                                | `POST /carrinhos`<br>с несуществующим `idProduto` | 400                                                                                                |
| CR-04 | Дубликат `idProduto` внутри одного запроса                                                                                       | -                                                                | `POST /carrinhos` с двумя одинаковыми `idProduto` | 400, Наличие дублирующих товаров не допускается                                                    |
| CR-05 | concluir-compra: товар остаётся списанным<br><br>Проверить `GET /produtos/{_id}`, что остаток не изменяется повторно             | Корзина активна (S1), известен остаток до                        | `DELETE /carrinhos/concluir-compra`               | 200, корзина -> S0, товар остаётся списанным                                                       |
| CR-06 | cancelar-compra возвращает товар на склад<br><br>Проверить `GET /produtos/{_id}`, что остаток пополнился на N                    | Корзина активна (S1), известен остаток до                        | `DELETE /carrinhos/cancelar-compra`               | 200, корзина -> S0, остаток товара вернулся к значению до создания корзины                         |
| CR-07 | Проверить правильность расчета `precoTotal`                                                                                      | Существует корзина, в ней есть несколько товаров в разном кол-ве | GET /carrinhos/{id}                               | 200, расчет выполнен верно<br><br>precoTotal = сумма (precoUnitario × quantidade) по всем позициям |
| CR-08 | concluir-compra, когда корзины вообще нет                                                                                        | Корзина в S0                                                     | `DELETE /carrinhos/concluir-compra`               | 200 (Для этого пользователя не найдено корзины), переход S0->S0                                    |
| CR-09 | cancelar-compra, когда корзины вообще нет                                                                                        | Корзина в S0                                                     | `DELETE /carrinhos/cancelar-compra`               | 200 (Для этого пользователя не найдено корзины), переход S0->S0                                    |

### Анализ граничных значений для `quantidade`

(N — остаток товара на складе)

| ID        | quantidade | Ожидаемый результат |
| --------- | ---------- | ------------------- |
| CR-BVA-01 | -1         | 400                 |
| CR-BVA-02 | 0          | 400                 |
| CR-BVA-03 | 1 (N >= 1) | 201                 |
| CR-BVA-04 | N - 1      | 201                 |
| CR-BVA-05 | N          | 201                 |
| CR-BVA-06 | N + 1      | 400                 |
